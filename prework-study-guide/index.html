HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Remember to check command palette for connection if lost.
  • Keep in mind the idea of semantic elements.

CSS

  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • This internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • The most common way to style a webpage is by using an external CSS style sheet.
  • When using an external style sheet, we link the file using a link element in the HTML's head.
  • Sometimes, we want to apply a rule to all the elements visible on the page. To do that, we use a special wild card symbol, or *.
  • The period preceding designates as a class selector.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git pull origin main: pull most recent origin main before working a new branch.
  • git add -A: stages your changes for pull requests.
  • git commit -m: commits changes staged for pull requests.
  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use.

JavaScript

  • var topics = "HTML, CSS, Git, JavaScript"; Here, we've created a variable called, topics. The topics variable contains a string that includes all of the topics that we are studying in this module: "HTML, CSS, Git, JavaScript". Notice that the string is contained within quotation marks, ". All strings are surrounded by quotation marks. Note that each line of JavaScript code ends with a semicolon.
